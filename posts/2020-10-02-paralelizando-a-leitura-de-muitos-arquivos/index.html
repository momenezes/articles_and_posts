Paralelizando a leitura de muitos arquivos no R

Acelerando a leitura de muitos de arquivos no R para posterior processamento.

Mario O. de Menezes https://momenezes.github.io/tutorials
10-02-2020

Introduo

O DATASUS, brao tecnolgico do Ministrio da Sade, disponibiliza uma vasta quantidade de informaes sobre diversos aspectos do Servio Unificado de Sade (SUS). Muitos destes arquivos so disponibilizados em um formato dbc, que um compressed DBF.

Estes arquivos so disponibilizados no site de FTP do DATASUS. Para fins deste pequeno experimento, todos os arquivos foram baixados e esto na mquina local.

O objetivo deste pequeno estudo experimentar tcnicas de paralelizao no R para acelerar a leitura dos arquivos.

Nem sempre operaes de I/O em paralelo trazem ganho de velocidade especialmente em se tratando de discos rgidos por rotao (os tradicionais), isto porque a operao de busca nestes discos pode incorrer em penalizao. Entretanto, neste experimento, obtivemos um bom resultado com a leitura em paralelo.

Carregando as bibliotecas

library(tidyverse)
library(read.dbc)
library(data.table)
library(microbenchmark)
library(parallel)

Listando todos os arquivos das APACS no diretrio

Todos os arquivos das APACs foram baixados do site FTP do DATASUS.

diretorio <- "~/datasets/datasus.gov.br/SIASUS/PA-LaudosDiversos/"

Selecionando os arquivos de Acre e Alagoras, de 2008 a 2012

PREF <- "PA"
# indique o estado ou "" para todos os estados
UF <- "(AC|AL)" 
MESES <- "(\\d\\d)"
ANO <- "(08|09|10|11|12)"
arquivos <- list.files(diretorio, paste0(PREF,UF,ANO,MESES,".*.dbc"))

No total, selecionamos 120 arquivos.

$ ls -ltr PA{AC,AL}{08,09,10,11,12}* | wc
    120    1080    7560

$ ls -ltr PA{AC,AL}{08,09,10,11,12}* | awk '{total += $5}; END {print total}'
523484797

Ou seja, para estes 120 arquivos, temos um total de 523.484.797 bytes, i.e., pouco mais de 500MB.

Nmero de cores disponveis para processamento paralelo. Esta uma mquina em uma nuvem privada.

(numCores <- detectCores())
[1] 32

A funo makeCluster utilizada para criar o nosso cluster virtual de processamento paralelo.

cluster <- makeCluster(numCores)

Criando uma lista com os nomes completos dos arquivos (com o diretrio).

fNames <- lapply(arquivos, function(x) {paste0(diretorio,x)})
arqs <- NULL

Leitura de 12 arquivos em modo serial

ptimes <- system.time(arqserial <- lapply(fNames[1:12], read.dbc))
ptimes[3]
elapsed 
  7.222

Limpando a memria

arqserial <- NULL
rm(arqserial)

Leitura de 120 arquivos em modo paralelo

A funo mclapply o equivalente paralelo da funo lapply que utilizamos acima. mclapply faz parte do pacote parallel muito conveniente para uma paralelizao trivial de operaes que envolvam listas.

library(parallel)
ptime <- system.time(arqs <- parallel::mclapply(fNames, read.dbc, mc.cores = numCores))
ptime[3]
elapsed 
 34.175

O tempo para leitura de apenas 12 arquivos foi 7.222 segundos, enquanto o tempo para a leitura de 120 arquivos em paralelo foi 34.175 segundos. Um excelente ganho de velocidade com a leitura paralela.

importante ressaltar que os arquivos destes dois estados so pequenos. Mesmo esta quantidade de arquivos (120) no comprometeu a memria da mquina. Nem sempre isso ocorre. preciso tomar cuidado!

Selecionando os arquivos de So Paulo, de 2010 a 2016

Os arquivos de So Paulo so muito maiores que os dos outros estados, de modo que constituem um desafio maior ainda para uma leitura em paralelo: otimizar esta leitura essencial!

PREF <- "PA"
# indique o estado ou "" para todos os estados
UF <- "SP" 
MESES <- "(\\d\\d)"
ANO <- "(10|11|12|13|14|15|16)"
arquivos <- list.files(diretorio, paste0(PREF,UF,ANO,MESES,".*.dbc"))

No total, selecionamos 133 arquivos.

$ ls -ltr PASP{10,11,12,13,14,15,16}* | wc
    133    1197    8610
$ ls -ltr PASP{10,11,12,13,14,15,16}* | awk '{total += $5}; END {print total}'
1.05298e+10

Como podemos ver, o tamanho dos arquivos muito maior; os 133 arquivos totalizam aproximadamente 10529800000 bytes, i.e., 10.529.800.000 bytes, ou 10GB.

Como o formato dbc um formato comprimido, quando fazemos a leitura cada data.frame fica bem maior.

f1 <- read.dbc(paste0(diretorio,arquivos[1]))
object.size(f1)
824834376 bytes

Aproximadamente 800MB.

Em disco, este arquivo tem aproximadamente 88MB:

$ ls -shc PASP1001.dbc
88M PASP1001.dbc

Ou seja, quase 10 vezes maior o tamanho em memria em relao ao tamanho em disco.

Criando uma lista com os nomes completos dos arquivos (com o diretrio).

fNames <- lapply(arquivos, function(x) {paste0(diretorio,x)})
arqs <- NULL

Leitura de 12 arquivos em modo serial

ptimes <- system.time(arqserial <- lapply(fNames[1:12], read.dbc))
ptimes[3]
elapsed 
845.107

Realmente, os arquivos do Estado de So Paulo so muito maiores do que dos outros estados.

Leitura de 12 arquivos em modo paralelo

A funo mclapply o equivalente paralelo da funo lapply que utilizamos acima. mclapply faz parte do pacote parallel muito conveniente para uma paralelizao trivial de operaes que envolvam listas.

library(parallel)

Vou usar 12 arquivos diferentes para no correr o risco de cache do SO.

ptime <- system.time(arqs <- parallel::mclapply(fNames[13:24], read.dbc, mc.cores = numCores))
ptime[3]
elapsed 
167.033

A leitura em paralelo teve uma reduo significativa no tempo; o speedup (\(\frac{temposerial}{tempoparalelo}\)) resulta em 5.06.

Apesar de esta mquina ter 32 cores e uma boa quantidade de memria RAM (128GB), ao tentar executar o mclapply com todos os 133 arquivos, travou a mquina. Estes 133 arquivos tem um tamanho em disco de quase 10GB; considerando uma razo de 10 vezes o tamanho em memria, chegaramos a quase 100GB; mais os buffers de leitura, etc., esgotamos a memria da mquina e vamos para o swap.

Assim, prudente executar em grupos de 10-12 arquivos para no incorrer no uso de swap, o que faz com que a mquina fique praticamente inacessvel.
